## 🛑 止损的艺术：什么时候该按下“重置键”？

Code is Cheap, Context is Expensive.

Vibe Coding 的最大优势是速度，最大劣势是“上下文腐烂” (Context Rot)。
当出现以下信号时，请立刻停止挣扎，按下New Chat：

1.  复读机现象：你向 AI 解释了 3 遍同一个逻辑，它依然写错。
    *   *原因*：当前的 Context 已经被之前的错误代码污染了，AI 陷入了局部最优解。
2.  打地鼠游戏：修好一个 Bug，冒出三个新 Bug。
    *   *原因*：代码结构已经极其脆弱，变成了“屎山”。
3.  甚至不需要理由：只要你觉得“越写越累”，就重开。

> 重构策略：
> 不要试图在原来的对话里“挽回局面”。
> 新建一个 Chat，把当前最核心的代码文件（如 `App.tsx`）粘贴进去，告诉它：“这是我现在的功能，请帮我重构/修复...”
> Vibe Coding 的重构成本极低——因为代码是 AI 写的，扔掉也不心疼。
