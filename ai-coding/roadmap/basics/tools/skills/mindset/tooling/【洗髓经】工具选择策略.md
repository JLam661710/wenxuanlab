## 🚀 关键时刻：什么时候该离开“新手村”？

从网页端工具 (Web AI) 到本地 IDE 的“惊险一跃”

Vibe Coding 的世界分为两层：
1.  云端沙箱层：Bolt.new, Lovable, MiaoDa (秒哒) —— 轻量、即开即用、零环境配置。
2.  本地工程层：Trae, Cursor, VS Code —— 专业、全能、完全控制。

很多非技术背景的开发者会问：“我什么时候需要从第一层跳到第二层？”

### 1. 觉醒时刻：触碰天花板 (The Ceiling Effect)
当你遇到以下任一情况时，说明你已经“毕业”了，网页端工具不再是你的帮手，而成了你的束缚：

*   数据持久化的焦虑：“我不希望用户的数据只存在浏览器的缓存里，我需要一个真正的数据库（MySQL/PostgreSQL）。”
*   复杂的后端逻辑：“我需要写一个定时任务，或者调用一个需要保密的支付接口（不能在前端暴露 Key）。”
*   项目的复杂度爆炸：“我的文件超过了 50 个，网页端的 AI 开始‘失忆’，记不住之前改了什么。”
*   部署与私有化：“我想把这个应用部署到我自己的服务器上，拥有独立的域名，而不是挂靠在别人的二级域名下。”

### 2. 迁移的代价：隐形的门槛 (The Hidden Cost)
IDE 不是免费的午餐，它对 Vibe Coder 提出了硬性要求。这不仅仅是软件的升级，更是“计算机素养”的补课：

*   硬件门槛：你不能再用 iPad 或 Chromebook 了。你需要一台正经的 Mac 或 Windows，最好有 16GB+ 内存。
*   操作门槛：你需要理解什么是“文件路径”、什么是“环境变量”、什么是“本地端口冲突”。
*   终端恐惧症 (CLI Phobia)：你必须面对那个黑底白字的命令行窗口。
    *   `npm install` (装依赖)
    *   `git push` (存代码)
    *   `python manage.py runserver` (跑服务)
    *   *这些不再是可选技能，而是生存技能。*

### 3. 决策矩阵：我该去还是留？

并不是所有人都需要迁移。“降维打击”有时候是更聪明的选择。

| 你的目标 (Goal) | 推荐方案 | 理由 |
| :--- | :--- | :--- |
| 一次性工具 (如：年会抽奖、Excel处理脚本) | 网页端 (Web AI) | 用完即走，无需环境，最快速度验证。 |
| 静态展示页 (如：个人简历、活动落地页) | 网页端 (Web AI) | 只有 HTML/CSS，不需要后端数据库。 |
| 内部小工具 (如：团队周报生成器) | 网页端 (Web AI) | 小圈子分享，容忍度高，能用就行。 |
| SaaS 产品 (如：记账软件、SaaS 服务) | 本地 IDE | 需要用户系统、支付、数据库，必须严肃对待。 |
| 长期维护项目 | 本地 IDE | 代码需要版本控制 (Git)，需要多人协作。 |

> 结论：网页端是“游乐场”，IDE 是“建筑工地”。
> 如果你只想搭积木，留在游乐场；如果你想盖大楼，欢迎来到工地——戴好安全帽（学习 CLI），这里虽然脏乱，但能建起摩天大楼。

### 4. 选题黑名单：新手千万别碰的“死亡禁区”

很多时候，不是你技术不行，也不是 AI 不行，而是你选错了题目。

对于 Vibe Coding 新手（尤其是非技术背景），请牢记：Web 应用是舒适区，其他领域可能是雷区。

#### ❌ 绝对不要碰的“雷区” (The Danger Zone)
1.  强平台依赖类：
    *   *案例*：“帮我写一个微信小程序 / 抖音小游戏 / 飞书插件。”
    *   *原因*：这些平台有极度复杂的鉴权机制 (Auth) 和私有 API，且文档更新极快，AI 的训练数据往往是过期的。
2.  硬件交互类：
    *   *案例*：“帮我写一个能控制家里空调开关的 App。”
    *   *原因*：涉及蓝牙、Wi-Fi 协议、物联网网关，调试极难（AI 看不到你的硬件）。
3.  逆向工程与黑灰产：
    *   *案例*：“帮我破解这个 VIP 视频网站 / 抢茅台脚本。”
    *   *原因*：不仅违法，而且涉及复杂的反爬虫对抗，AI 无法处理动态加密参数。
4.  大型系统重构：
    *   *案例*：“把这个 5 万行代码的 Java 项目重构为 Python。”
    *   *原因*：超出了 Context Window（上下文窗口）的极限，AI 会顾头不顾尾。

#### ✅ 强烈推荐的“舒适区” (The Comfort Zone)
1.  纯 Web 应用 (SPA)：
    *   *案例*：个人博客、Todo List、番茄钟、记账本、数据可视化大屏。
    *   *优势*：浏览器环境标准统一，AI 训练数据最丰富。
2.  数据处理脚本：
    *   *案例*：Excel 格式转换、PDF 提取文字、批量重命名文件。
    *   *优势*：逻辑单一，输入输出明确，Python 生态极强。
3.  基于标准 API 的工具：
    *   *案例*：调用 OpenAI API 做翻译工具、调用 Unsplash API 做壁纸下载器。
    *   *优势*：接口文档公开标准，AI 能精准写出调用代码。

4.  AI Native 应用开发 (Advanced)：
    *   *案例*：给智能体配置 Tool/Skill，集成 RAG 知识库，设计多智能体协作流程。
    *   *优势*：这是 AI 最懂的领域（自我指涉），且是未来的核心方向。用 AI 写 AI，是 Vibe Coding 的最高境界。

> 原则：
> 越封闭的系统，越难 Vibe；越开放的标准，越容易 Vibe。
> 新手的第一行代码，应该运行在浏览器里，而不是微信或者单片机里。
