## 🤯 进阶陷阱：当 Vibe Coder 试图开发 AI 应用

“用 AI 写 AI”的递归诅咒

很多学员在做完 Todo List 后，雄心勃勃地想做一个“自己的 ChatGPT”。
结果往往撞得头破血流。

### 1. 维度的跃迁
开发一个“记账本”和开发一个“AI 对话助手”，在难度上不是一个数量级的。
*   普通应用：处理确定的数据（增删改查）。
*   AI 应用：处理概率性的数据（Token 流），并涉及密钥管理、网络代理、上下文窗口、流式传输 (SSE) 等一系列复杂工程问题。

### 2. 常见的“鬼打墙”
*   API Key 泄露：直接把 `sk-xxxx` 写在前端代码里，导致 Key 被封禁。
*   代理地狱：本地能跑通，部署上线后因为服务器网络问题调不通 OpenAI。
*   过度工程化：为了隐藏 Key，被 AI 忽悠着去配置 Cloudflare Workers 或 Netlify Functions，结果陷入运维的泥潭。

> 建议：
> 如果你是 Vibe Coding 新手，请先避开 AI 应用开发，或者直接使用 Vercel AI SDK 等封装好的成熟库。
> 不要试图从零手写 HTTP 请求去调用 LLM，那是造轮子，不是 Vibe。
