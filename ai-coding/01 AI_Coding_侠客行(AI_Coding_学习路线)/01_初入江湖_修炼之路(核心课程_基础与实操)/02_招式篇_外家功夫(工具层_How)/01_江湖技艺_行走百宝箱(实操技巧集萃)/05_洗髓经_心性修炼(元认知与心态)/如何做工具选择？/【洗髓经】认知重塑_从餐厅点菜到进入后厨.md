# 认知重塑：从“餐厅点菜”到“进入后厨”

## 1. 核心隐喻：餐厅与后厨

在 Vibe Coding 的前三个阶段（No-Code 阶段），学员们就像是在一家米其林餐厅用餐。
*   体验：环境优雅（界面好看），服务周到（操作简单），只需要指着菜单说“我要这个”（Prompting），美味佳肴（App）就会端上来。
*   局限：
    *   你无法要求厨师把牛排里的黑胡椒换成白胡椒，因为菜单上没写。
    *   如果菜咸了，你除了投诉（重新生成），别无他法。
    *   最重要的是，你永远不知道这道菜是怎么做出来的。

从第四课开始，我们要带大家“推开后厨的门”（IDE & CLI）。
*   冲击：后厨是嘈杂的、有油烟的、到处是原材料的（代码、文件、报错信息）。
*   权力：但在这里，你可以决定放多少盐，用什么火候。你可以创造菜单上没有的菜。
*   目标：不是为了让每个人都成为厨师长，而是为了让大家知道，当你想吃一口“特制菜”时，该去哪里找调料。

## 2. 掀开“头盖骨”：恢复工具的可解释性

你提到的“掀开 No-Code 工具的头盖骨”，本质上是媒介现象学的一次实践。我们需要揭示被封装在漂亮 UI 之下的物质性结构：

### 2.1 文件系统（The Skeleton）
*   No-Code 视角：只有一个“项目名称”。
*   IDE 视角：
    *   `index.html`：这是骨架。
    *   `style.css`：这是皮肤。
    *   `app.py` / `main.js`：这是大脑和神经。
    *   教学点：让学员亲手点开这些文件，哪怕看不懂代码，也要让他们看到“哦，原来我的网页就是这几个文件组成的，不是魔法。”

### 2.2 环境变量（The Vault）
*   No-Code 视角：一个输入 Key 的文本框，输完就看不到了。
*   IDE 视角：
    *   `.env` 文件：一个不起眼的文本文件，却掌管着通往世界的钥匙。
    *   教学点：通过“小偷演示”（见架构篇），让学员明白为什么这个文件必须被加入 `.gitignore`（隐身斗篷）。

### 2.3 API 交互（The Nervous System）
*   No-Code 视角：点击“连接插件”。
*   IDE 视角：
    *   HTTP Request：一次明确的握手。
    *   JSON Response：一次明确的反馈。
    *   教学点：打破“智能”的幻觉，还原为“数据交换”的本质。

## 3. 突破成长上限

*   现状：只用 No-Code 工具，学员的技能上限就是“工具功能的上限”。工具没做这个功能，你就做不出来。
*   突破：学会连接 API，学会使用 IDE，学员的上限就变成了“想象力的上限”。
*   教学话术：
    > “同学们，前三节课我们是在‘骑共享单车’，扫码即走，很方便，但你只能去地图上画了线的地方。
    > 从今天开始，我们要学习‘造越野车’。虽然一开始要拧螺丝很麻烦，但造好之后，你可以去任何没有路的地方。”

通过这种去魅（Disenchantment）的过程，我们帮助学员克服对技术的恐惧，建立起对底层逻辑的掌控感。
