## 战略定力：写 PRD 一定不能快

我们在实践中反复验证了一个真理：**前面写 PRD 越详细、越打磨，后面开发的时候就越顺畅。**（Slow is Smooth, Smooth is Fast）

但是，“写 PRD 一定不能快”只是一句正确的废话。如果不提供可操作的方案，它就是无效的建议。

**如何在写 PRD 的过程中真正“慢下来”？**
核心在于摒弃“指望输入一句话就得到完美 PRD”的幻想，转而建立一套**六维降速模型**：

### 1. 观念（Mindset）：PRD 是核心资产
*   不要把 PRD 当作一次性的“提示词”，它是你在 AI Coding 时代最重要的**核心资产**。
*   它值得你像打磨代码一样，逐字逐句地去打磨、去迭代、去版本管理。

### 2. 心态（Attitude）：放弃“一轮直出”
*   彻底忘记 Vibe Coding 宣传的“秒生成”虚假承诺。
*   不急功近利，接受**“对话—反馈—修正”**的漫长循环。好的 PRD 是“聊”出来的，不是“生成”出来的。

### 3. 认知（Cognition）：预期管理的能力
*   建立所谓的 **Taste（鉴赏力）** 和 **Control（控制力）**。
*   在足量的实践后，你一眼就能看出 AI 生成的 PRD 哪里在“胡说八道”，哪里在“敷衍了事”，并能精准地叫停。

### 4. 技巧（Technique）：上下文管理工程
*   写 PRD 的难点不在于文笔，而在于**Context Management（上下文管理）**。
*   如何让 AI 记住之前的设定？如何处理冲突的需求？如何在长对话中保持逻辑一致性？这是真正的技术活。

### 5. 方法（Method）：回归传统产品力
*   不要只会对着 AI 聊天。回归传统的产品方法论：
    *   **问卷/表单**：结构化收集需求。
    *   **访谈/用户研究**：深度挖掘痛点。
    *   **竞品分析**：寻找参考坐标。

### 6. 工具（Tool）：善用辅助系统
*   **主动式 AI 访谈者**：让 AI 反过来向你提问，倒逼你思考。
*   **AI IDE / Chatbot**：利用工具的 Memory 功能。
*   **Agent Workflow**：构建自动化的 PRD 审查工作流。
