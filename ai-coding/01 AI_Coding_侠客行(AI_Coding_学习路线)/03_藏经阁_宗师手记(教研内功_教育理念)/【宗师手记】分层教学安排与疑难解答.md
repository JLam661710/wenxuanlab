# 分层教学安排与疑难解答

## 1. 疑难解答与选型建议

### Q1: Tavily 是什么？为什么要选它？有没有中文替代品？
*   **Tavily 是什么**：它是**“专门给 AI 用的搜索引擎”**。
    *   普通 Google/百度搜索结果是给**人**看的（有广告、有无关信息）。
    *   Tavily 的结果是经过清洗的、纯净的文本，AI 读起来不费劲，且 Token 消耗少。
*   **为什么要选它**：
    *   **极简**：API 设计极其友好，一行代码就能拿到高质量结果。
    *   **免费额度**：每个月有 1000 次免费调用，足够教学用。
*   **中文替代品**：
    *   **博查 (Bocha)**：国内做得很好的 AI 搜索 API，对中文源支持更好，也有免费额度。
    *   **教学策略**：可以把 Tavily 作为“国际标准”演示，把博查作为“国内实战”推荐，让学员二选一，这也顺便教了**“技术选型”**的概念。

### Q2: 天气 API 选哪个？
*   **首选：OpenWeatherMap**
    *   **优点**：国际最通用，文档极其标准（教科书级别），免费版够用。
    *   **缺点**：全是英文文档（正好可以锻炼学员用翻译插件或 AI 读文档的能力）。
*   **备选：高德地图开放平台 / 和风天气**
    *   **优点**：中文文档，数据更准（国内）。
    *   **缺点**：注册认证流程（实名认证）可能比较繁琐，容易劝退。
*   **建议**：教学用 **OpenWeatherMap**，因为它无需实名认证，邮箱注册即用，门槛最低。

### Q3: 飞书机器人/多维表格没操作过，怎么演示？
*   **直观展示法**：
    *   **录屏/动图**：不要只放代码。先放一个 GIF —— 你的代码一点运行，飞书群里“叮”的一声弹出一行字。这种**“魔法感”**最能抓人。
    *   **类比**：
        *   **机器人** = “你雇了个哑巴发报员，给他一张纸条（JSON），他就跑去群里贴一张大字报。”
        *   **多维表格** = “你雇了个会计，给他一张发票（JSON），他就帮你填到 Excel 表格的下一行。”

### Q4: “无 API 挑战”放在哪里？
*   **定位**：这是一个**“彩蛋”**或**“危机处理”**章节。
*   **放置位置**：建议放在**所有 API 课程结束之后**，作为**“加餐课”**。
*   **引入话术**：
    *   “同学们，到现在为止，世界对你们来说都是美好的（都有 API）。但是，如果有一天，老板让你去抓一个没有 API 的网站数据，你该怎么办？这节课我们来学点‘黑魔法’。”
*   **融合方式**：不要把它当成主线，以免增加难度焦虑。把它包装成一个**“破解难题的锦囊”**。

## 2. 分阶段分层级教学安排 (Syllabus)

基于学员“熟悉 AI 但不熟悉工程”的特点，我们将课程分为三个阶段：**舒适区 -> 学习区 -> 挑战区**。

### 第一阶段：舒适区（建立连接，巩固信心）
*   **目标**：用代码控制他们已经熟悉的 AI。
*   **内容**：
    1.  **大模型 API (OpenAI/DeepSeek)**：
        *   重点：流式输出（Streaming）的视觉效果。
        *   实操：做一个简易的命令行聊天机器人。
    2.  **生图 API (SiliconFlow/Midjourney)**：
        *   重点：从文本到图片的转换，文件的保存。
        *   实操：输入一句话，自动下载一张图到桌面。

### 第二阶段：学习区（突破盲区，掌握结构）
*   **目标**：接触从未听过的工具，理解“结构化数据”。
*   **内容**：
    3.  **天气 API (OpenWeatherMap)**：
        *   **痛点**：JSON 层级多，难找。
        *   **教学**：使用 JSON Formatter 工具，手把手教“剥洋葱”。
    4.  **AI 搜索 API (Tavily/博查)**：
        *   **痛点**：Tavily 是啥？
        *   **教学**：对比演示。左边放百度搜索结果（乱），右边放 Tavily 返回结果（纯净）。
        *   **实操**：写一个“全网热点聚合器”。

### 第三阶段：挑战区（工程落地，闭环思维）
*   **目标**：从“玩玩”变成“生产力”。
*   **内容**：
    5.  **飞书/钉钉机器人 (Webhook)**：
        *   **预演**：先看效果视频。
        *   **实操**：把第二阶段抓到的“全网热点”或“天气预警”，自动发到群里。
    6.  **多维表格 (Database)**：
        *   **概念**：这是你的**云端数据库**。
        *   **实操**：把每次生成的图片记录、天气记录，自动存表。
    7.  **（加餐）无 API 挑战**：
        *   **定位**：选修课。
        *   **实操**：简单的网页标题抓取。

## 3. 教学物料准备清单

为了降低学员的认知负荷，让他们在“动手写代码”之前先建立“心理预期”，你需要准备一套**“所见即所得”**的视觉化辅助物料。

### 3.1 视觉化预演（让效果“肉眼可见”）
*   **Gif 动图 / 短视频（必杀技）**：
    *   **飞书/钉钉推送**：录制屏幕，左边是代码编辑器点击“运行”，右边是飞书客户端瞬间弹出消息。这种**跨软件的联动**最能带来“黑客既视感”。
    *   **流式输出对比**：做一个对比视频。左边是“转圈等待30秒一次性吐出结果”，右边是“打字机效果逐字蹦出”。让学员直观感受到 streaming 的必要性。
*   **对比图（Before & After）**：
    *   **数据清洗**：左边是一张乱糟糟的网页 HTML 源码截图（标红字：人类无法阅读），右边是 Tavily API 返回的清爽 JSON 截图（标绿字：AI 的最爱）。
    *   **多维表格**：左边是空荡荡的表格，右边是代码运行后自动填满数据的表格。

### 3.2 半成品代码包（Starter Kit）
不要让学员从空白文件开始写（Blank Page Syndrome），这会带来巨大的压力。
*   **填空题式代码**：
    ```python
    # 你的 API Key 填在这里
    API_KEY = "________" 

    # 这是一个发送请求的函数，我已经帮你写好了基础设置
    def get_weather(city):
        url = "https://api.openweathermap.org/..."
        # TODO: 这里少了一行代码来发送请求，请你补上
        # 提示：response = requests.get(...)
    ```
*   **依赖环境一键包**：提供一个 `requirements.txt`，里面只有最核心的库（`requests`, `openai`），避免学员在安装环境上浪费半小时。

### 3.3 “救命锦囊”文档（Cheat Sheet）
*   **常见报错翻译字典**：
    *   `401 Unauthorized` -> “你的钥匙（API Key）错了或过期了。”
    *   `404 Not Found` -> “你地址写错了，或者城市名字拼错了。”
    *   `KeyError` -> “你试图在柜子里找一个不存在的标签。”
*   **Prompt 咒语卡**：
    *   提供一套标准话术给学员复制：“我的代码报错了，错误信息是...，请用中文解释原因并给出修改后的代码。”

### 3.4 真实世界的数据样本（Mock Data）
*   **离线 JSON 文件**：
    *   准备几个 `weather_sample.json`, `search_result_sample.json`。
    *   **用途**：万一教室断网了，或者学员的 API Key 额度用光了，他们依然可以用这些本地文件来练习“解析数据”的代码。这叫**“教学灾备方案”**。
