# 教学案例设计：API 进阶与实战闭环

## 1. 核心教学案例库（从入门到综合）

根据之前的讨论，我们将教学案例分为六个层级，旨在循序渐进地构建学员的技能树。

### 01. 大模型 API (Text Generation) —— 入门与对话
*   **目标**：建立信心，理解 AI 交互基础。
*   **核心点**：
    *   **Streaming（流式传输）**：解释为什么字是一个个蹦出来的（提升用户体验）。
    *   **Context（上下文）**：`messages` 数组的管理，理解 `system` 设定与 `user` 历史记录的作用。

### 02. 天气查询 API (Structured Data) —— 数据结构特训
*   **目标**：掌握 JSON 解析与参数传递。
*   **核心点**：
    *   **Query Parameters**：理解 `?city=Beijing` 这种传参方式。
    *   **Deep Nested JSON**：练习提取深层数据（如 `data.daily[0].temp.max`），克服对复杂结构的恐惧。

### 03. 生图模型 API (Image Generation) —— 多模态体验
*   **目标**：处理非文本数据（文件）。
*   **核心点**：
    *   **Input**：Base64 编码的使用（如何上传参考图）。
    *   **Output**：URL 处理与文件下载（File I/O），将网络上的图片保存到本地硬盘。

### 04. Tavily Search API (Info Retrieval) —— 真实数据清洗
*   **目标**：处理海量、非结构化数据。
*   **核心点**：
    *   **List Filtering**：从 10 个结果中筛选出有用的信息。
    *   **Data Cleaning**：清洗网页抓取的乱码文本。
    *   **RAG 雏形**：为后续结合大模型做准备。

### 05. 飞书/钉钉机器人 & 多维表格 (Action & Push) —— 成果落地
*   **目标**：从“获取数据”转向“发送数据/触发动作”。
*   **场景 A：群机器人（最简单）**
    *   **操作**：获取一个 Webhook URL，发送一个 JSON `{"msg_type": "text", "content": {"text": "你好"}}`。
    *   **爽点**：代码一跑，手机立刻响。极强的物理反馈。
*   **场景 B：多维表格写入（进阶）**
    *   **操作**：将 Tavily 搜到的新闻，自动填入飞书多维表格的一行。
    *   **价值**：这是自动化办公（RPA）的核心，非常实用。

### 06. “无 API” 挑战 (The "No-API" Reality) —— 真实世界的残酷
*   **目标**：理解当没有现成 API 时该怎么办（爬虫/网页解析概念）。
*   **操作演示**：
    *   **场景**：想获取某新闻网站的头条，但它没有 API。
    *   **方法**：使用简单的 HTML 解析库（如 `BeautifulSoup` 或 AI 辅助解析）。
    *   **教学点**：查看网页源代码 -> 找到标题所在的标签（如 `<h1>`） -> 提取文字。
    *   **意义**：祛魅“网页”，理解网页本质上也是一种（乱糟糟的）结构化数据。

## 2. 综合大作业：全栈 Agent 雏形

将上述模块串联，完成一个完整的自动化工作流：

1.  **Tavily** 搜索“今日 AI 新闻”。
2.  **大模型** 总结新闻摘要，并翻译成中文。
3.  **生图模型** 根据新闻标题生成一张配图。
4.  **飞书机器人** 将【摘要 + 配图】发送到公司群。
5.  **多维表格** 归档这条新闻记录。

**这就是一个价值百万的 Agent 的最小可行性产品（MVP）。**
