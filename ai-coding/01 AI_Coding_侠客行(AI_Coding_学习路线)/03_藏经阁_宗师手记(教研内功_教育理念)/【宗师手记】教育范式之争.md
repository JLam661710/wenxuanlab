## 🔮 教育范式之争：收敛与发散的博弈

AI 时代的课程设计哲学

在设计 Vibe Coding 课程时，我们面临一个终极难题：
“是教大家复现经典项目（如番茄钟），还是让大家自由开发个性化需求？”

这背后是两种教育范式的碰撞：
*   旧范式 (The Old Way)：工业化、标准化、结果收敛（所有人都做出一样的番茄钟）。
    *   *优点*：容易教学，路径清晰，保证成功率。
    *   *缺点*：无聊，扼杀创造力，学生做完就扔。
*   新范式 (The New Way)：个性化、发散性、结果未知（每个人做自己想做的）。
    *   *优点*：热情高涨，解决真实痛点，Vibe 十足。
    *   *缺点*：需求过于发散，技术难度不可控，教学资源无法覆盖。

### 1. 现状：青黄不接的探索期
我们正处于一个“旧的已死，新的未生”的尴尬阶段。
*   完全沿用旧模式，会被 AI 时代抛弃（AI 一秒生成标准代码，复现毫无意义）。
*   完全采用新模式，会导致“教学灾难”（学生需求五花八门，助教疲于奔命）。

### 2. 破局：半开放式命题 (Semi-Open Structure)
未来的 Vibe Coding 教育，应当采用“骨架收敛，血肉发散”的策略。

*   骨架（收敛）：
    *   规定核心技术栈（如 React + Tailwind）。
    *   规定功能复杂度（如“必须包含增删改查”）。
    *   *例如*：“本周任务是开发一个【个人效率工具】。”
*   血肉（发散）：
    *   场景自定义：你可以做番茄钟，他可以做背单词卡片，她可以做冥想计时器。
    *   风格自定义：你可以做赛博朋克风，他可以做极简主义风。
    *   Prompt 自定义：鼓励用不同的 Prompt 路径达到相同的技术目的。

> 结论：
> 最好的 Vibe Coding 课程，不是教你“如何画一只标准的马”，而是教你“如何握住画笔”，然后鼓励你画出你心中的独角兽。
> 我们不需要更多的番茄钟，我们需要的是被 AI 武装起来的、鲜活的创造力。
