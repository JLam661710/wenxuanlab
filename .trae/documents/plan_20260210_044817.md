### 核心问题诊断
您项目中使用了极具特色的“武侠风”长中文目录名（如 `/ai-coding/roadmap/...`）。
*   **优点**：结构清晰，像大纲一样，且直接决定了网站菜单的显示文字。
*   **缺点**：由于 VitePress 默认使用文件夹名作为 URL，导致**文件系统路径极长**，超过了许多部署环境的限制，从而导致部署失败。

### 解决方案：借尸还魂（物理改名，逻辑保留）
我们可以把“文件存储名”和“网站显示名”解耦。就像给图书馆的书架贴上简短的编号（A区、B区），但在目录卡片上依然写着完整的书名。

**具体步骤如下：**

#### 1. 物理层：缩骨功（重命名文件夹）
我们将把所有冗长的中文文件夹名称，修改为简短、安全的英文编号代码。
*   **原样（太长）：** `/ai-coding/roadmap/`
*   **修改后（简短）：** `/ai-coding/sec-01/`

我们会按照层级结构，系统地将它们重命名为 `sec-01` (第一章), `part-01` (第一节), `mod-01` (第一块) 等。

#### 2. 逻辑层：易容术（修改配置文件）
在 `.vitepress/config.mts` 文件中，我们将手动“告诉”网站：虽然文件放在了 `sec-01`，但请你在菜单上依然显示为“01 AI_Coding_侠客行(AI_Coding_学习路线)”。

*   **修改前：**
    ```javascript
    { text: "01 AI_Coding_侠客行...", link: "/...长路径.../" }
    ```
*   **修改后：**
    ```javascript
    { text: "01 AI_Coding_侠客行...", link: "/ai-coding/sec-01/..." } // text 不变，link 变短
    ```

#### 3. 链接层：经脉重续（修复引用）
我会扫描所有 Markdown 文件，自动把里面引用的图片链接和文章跳转链接，从旧的长路径更新为新的短路径，确保点击跳转和图片显示一切正常。
