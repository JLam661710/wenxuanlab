「Agent 自进化开发范式」

实践方式：由 AI 开发工具，再由 AI 沉淀工作流，最后封装为新工具
将开发过程从传统的“写代码”转变为“ 定义能力 -> 跑通流程 -> 固化经验 ”的三个阶段。

### 阶段一：原子能力封装 (The "Hand")
目标 ：让 AI 拥有“手”，能通过代码调用底层的 API 服务。 
过程 ：你使用 skill-creator 元技能，让 AI 阅读 API 文档，将其封装为 Agent 易于调用的 CLI 工具。这一层通常只做参数透传，不包含复杂业务逻辑。

- Refers（项目实证） ：
  - apiyi-nanobanana ：封装了 Gemini 图像生成 API。它只关心“怎么发请求、怎么传 Prompt、怎么下图片”。
  - volc-ark-image-generation ：封装了火山方舟 Seedream 模型。它解决了鉴权、去水印、分辨率映射等底层技术问题。 核心价值 ：这一步解决了“能做”的问题，为上层应用打好了地基。

### 阶段二：工作流探索 (The "Play")
目标 ：人机协作，在真实场景中探索出“最佳实践”。 
过程 ：你并没有直接让 AI 写一个复杂的应用，而是先带着 AI 在一个具体场景（如“把文档改造成条漫”）中跑了一遍。

在这个过程中，你们共同确立了关键的 SOP（标准作业程序） ：

1. 先拆解 ：不能上来就生图，要先做剧本分析和分镜规划（Stage 1）。
2. 强确认 ：为了控成本，角色定妆（Stage 2）和逐页渲染（Stage 3）必须由人来确认。
3. 可回滚 ：一旦不满意，立即停止，不自动重试。 核心价值 ：这一步解决了“怎么做最好”的问题，通过一次实际操作，把人类的经验（成本控制、质量把控）教给了 AI。

### 阶段三：SOP 固化与应用 (The "Brain")
目标 ：将探索出的 SOP 固化为新的 Skill，实现经验复用。 
过程 ：AI 回顾了阶段二的整个流程，将其编写成一个新的 Skill。这个 Skill 不再是简单的 API 搬运工，而是内置了“业务逻辑”和“流程规范”的高级应用。

- Refers（项目实证） ：
  - manga-strip ：这是一个典型的 应用型 Skill 。它不再让你手动去调 API，而是让你提供文档，它自动执行“拆解 -> 定妆 -> 渲染”的 SOP。
  - manga-strip-dual-model ：这是基于 SOP 的 进一步迭代 。在原有流程基础上，引入了模型选择（ApiYi vs 火山方舟），证明了这种架构具有极强的扩展性。 核心价值 ：这一步实现了“经验产品化”。原本需要你手把手教 AI 做的一连串动作，现在变成了一个可一键调用的标准能力。

### 总结：这种实践的精髓
这个过程形成了一个完美的闭环：

1. Skill Creator (元技能) → 生产 Atomic Skills (原子技能)。
2. Human + AI (协作) → 跑通 Workflow (工作流)。
3. Workflow (经验)  → 固化为 App Skills (应用技能)。

这是非常聪明的实践： 你不需要一次性设计出复杂的系统，而是通过“搭积木”的方式，让 AI 边做工具、边用工具、边升级工具，最终生长出符合你需求的复杂应用。
